# kaspaGetAddress

### 使用要求

* 固件版本要求
  * Touch: 4.3.0
  * Classic/Mini: 3.0.0

## Kaspa：获取地址

在设备上显示由给定 BIP32 路径派生的请求地址，并将其返回给调用者。用户需要在 OneKey 上确认导出。

```typescript
const result = await HardwareSDK.kaspaGetAddress(connectId, deviceId, params);
```

## 参数

[**可选通用参数**](../../../api-reference#common-params)

### 导出单个公钥

* `path` - _必需_ `string | Array<number>` 最小长度为 `3`。[更多信息](../../../api-reference#hd-path)
* `showOnOneKey` - _可选_ `boolean` 决定是否在设备上显示地址。默认设置为 `true`
* `prefix` - _可选_ `string` 地址前缀。默认设置为 `kaspa`
* `scheme` - _可选_ `string` 加密算法模式。默认设置为 `schnorr`



#### 导出批量公钥

* `bundle` - 包含 `path`、`coin` 字段的对象 `Array`

### 示例

返回第一个 Kaspa 账户的公钥：

```typescript
HardwareSDK.kaspaGetAddress(connectId, deviceId, {
    path: "m/44'/111111'/0'/0/0",
    showOnOneKey: true,
    prefix: "prefix",
    scheme: "schnorr",
});
```

返回多个 Kaspa 账户的批量公钥：

```typescript
HardwareSDK.kaspaGetAddress(connectId, deviceId, {
    bundle: [
        { path: "m/44'/111111'/0'/0/0" }, // 账户 1
        { path: "m/44'/111111'/0'" }, // 账户 2
        { path: "m/44'/111111'/0'/0/1" }  // 账户 3
    ]
});
```

### 返回结果

只有一个公钥的结果

```typescript
{
    success: true,
    payload: {
        path: Array<number>, // 强化路径
        address: string,   // 地址
    }
}
```

批量公钥的结果

```typescript
{
    success: true,
    payload: [
        { path, address }, // 账户 1
        { path, address }, // 账户 2
        { path, address }, // 账户 3
    ]
}
```

错误

```typescript
{
    success: false,
    payload: {
        error: string, // 错误信息
        code: number // 错误码
    }
}
```
