# tronSignMessage

## TRON：签名消息

请求设备使用由给定 BIP32 路径派生的私钥对消息进行签名。

```typescript
const result = await HardwareSDK.tronSignMessage(connectId, deviceId, params);
```

### 参数

[**可选通用参数**](../../../api-reference#common-params)

* `path` - _必需_ `string | Array<number>` 最小长度为 `3`。[更多信息](../../../api-reference#hd-path)
* `messageHex` - _必需_ `string` 十六进制格式的待签名消息
* `messageType` - _可选_ `string` TRON 消息签名算法类型。推荐使用 `'v2'` 以获得更好的兼容性和安全性。允许值：`'v1' | 'v2'`。除非有遗留系统约束，否则请使用 `'v2'`。

### 示例

```typescript
// 原始消息
const message = "Hello World";

// 转换为十六进制
const messageHex = Buffer.from(message).toString('hex');

HardwareSDK.tronSignMessage(connectId, deviceId, {
    path: "m/44'/195'/0'/0/0",
    messageHex,
    messageType: 'v2'
});
```

返回结果

```typescript
{
    success: true,
    payload: {
        signature: string;
        address: string;
    }
}
```

错误

```typescript
{
    success: false,
    payload: {
        error: string, // 错误信息
        code: number // 错误码
    }
}
```
