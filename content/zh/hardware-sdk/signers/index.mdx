# 交易签名向导

面向“从初始化到拿到签名”这一条完整链路，按链拆分为可复制的分步示例。阅读前请先完成 [请求基础](../api-reference#request-basics)。

## 结构

- 前置条件：固件/SDK 版本、是否需要设备确认、必需的传输方式
- 步骤式流程：连接设备 → 选择账户路径 → 构造请求 → 设备确认 → 结果校验
- 最小可运行示例：包含事件监听、请求参数、结果校验
- 常见问题：路径错误、拒签/超时、网络或数据格式问题

## 可用向导

- [EVM Signer](./evm) — EIP-1559/Legacy 交易与签名校验
- [BTC Signer](./btc) — UTXO 构造、refTx 处理、签名输出
- [Solana Signer](./solana) — 固定路径、序列化原始交易签名
