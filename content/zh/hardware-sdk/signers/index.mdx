# 交易签名向导

面向“从初始化到拿到签名”这一条完整链路，按链拆分为可复制的分步示例。阅读前请先完成 [请求基础](../core-api-guide#request-basics)。

## 结构

- 前置条件：固件/SDK 版本、是否需要设备确认、必需的传输方式
- 步骤式流程：连接设备 → 选择账户路径 → 构造请求 → 设备确认 → 结果校验
- 最小可运行示例：包含事件监听、请求参数、结果校验
- 常见问题：路径错误、拒签/超时、网络或数据格式问题

## 快速检查清单（开始前必看）

- 环境与版本：使用最新固件，SDK/Bridge 版本对齐；按平台选择对应传输指南。
- 路径：按链使用正确 HD Path，注意硬化/非硬化（详见各链签名页）。
- 会话与钱包：多数链方法需 `connectId + deviceId`（BLE 必带 `deviceId`）；设备重置后 `deviceId` 会变化；使用隐藏钱包时携带 `passphraseState`。
- UI：订阅 `UI_EVENT`，在 PIN/Passphrase 请求时调用 `uiResponse`。
- 结果校验：先判 `success`，再用链工具库校验返回的地址/签名。
- 安全：遇到异常提示可选择拒签；设备在 Bootloader 时应直接失败。
