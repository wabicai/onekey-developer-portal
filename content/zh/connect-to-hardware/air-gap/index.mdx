# Air-Gap（离线二维码签名）

使用 UR 编码二维码构建"设备保持离线"签名流程的简明概述。

## 工作原理

- 构建签名请求 UR 数据包（按链类型），然后渲染为动画二维码帧。
- 设备扫描二维码，在设备上验证详情，并离线签名。
- 设备以动画二维码形式显示签名/结果 UR。
- 应用扫描响应，验证后通过您的网络堆栈提交/广播。

请参阅下方的快速开始和 API 参考获取具体类型和示例。

## 示例和演示

- 快速开始：[概述](quick-start)，按链：[EVM](quick-start/evm)、[BTC](quick-start/btc)、[Solana](quick-start/solana)、[TRON](quick-start/tron)
- React Native 演示：[GitHub Air-Gap Demo](https://github.com/OneKeyHQ/hardware-js-sdk/tree/onekey/packages/connect-examples/react-native-demo/air-gap)
