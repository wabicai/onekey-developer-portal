# nostrGetPublicKey

### 使用要求

* 固件版本要求
  * Touch: 4.6.0
  * Classic/Mini: 3.5.0
  * Classic1s: 3.6.0

## Nostr：获取公钥

获取由给定 BIP32 路径派生的 BIP32 扩展公钥。用户会看到所请求密钥的描述，并被要求确认导出。

```typescript
const result = await HardwareSDK.nostrGetPublicKey(connectId, deviceId, params);
```

## 参数

[**可选通用参数**](../common-params)

### 导出单个公钥

* `path` - _必需_ `string | Array<number>` 最小长度为 `3`。[更多信息](../path)
* `showOnOneKey` - _可选_ `boolean` 决定是否在设备上显示地址。默认设置为 `true`



#### 导出批量公钥

* `bundle` - 包含 `path`、`coin` 字段的对象 `Array`

### 示例

返回第一个 Nostr 账户的公钥：

```typescript
HardwareSDK.nostrGetPublicKey(connectId, deviceId, {
    path: "m/44'/1237'/0'/0/0",
    showOnOneKey: true,
});
```

返回多个 Nostr 账户的批量公钥：

```typescript
HardwareSDK.nostrGetPublicKey(connectId, deviceId, {
    bundle: [
        { path: "m/44'/1237'/0'/0/0" }, // 账户 1
        { path: "m/44'/1237'/0'" }, // 账户 2
        { path: "m/44'/1237'/0'/0/1" }  // 账户 3
    ]
});
```

### 返回结果

只有一个公钥的结果

```typescript
{
    success: true,
    payload: {
        path: Array<number>, // 强化路径
        publickey: string,   // 公钥
        npub: string,        // npub
    }
}
```

批量公钥的结果

```typescript
{
    success: true,
    payload: [
        { path, publickey, npub }, // 账户 1
        { path, publickey, npub }, // 账户 2
        { path, publickey, npub }, // 账户 3
    ]
}
```

错误

```typescript
{
    success: false,
    payload: {
        error: string, // 错误信息
        code: number // 错误码
    }
}
```
